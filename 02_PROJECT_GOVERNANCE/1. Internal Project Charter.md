## 🌟 2. 範例：自動化報表工具 (Internal Tool)

假設場景：你是公司的工程師，業務部門每天都在哀嚎，說從舊 ERP 抓資料整理報表很花時間。你決定寫個小工具救他們。

**專案名稱：** Sales Daily Report Auto-Bot (業務日報自動化機器人)
**開發負責人：** Gemini
**需求單位：** 業務部 (Manager: Alice)
**日期：** 2025/12/30

#### 1. Background & Pain Points (痛點)

* **現狀：** 業務助理每天早上需花 40 分鐘，從舊 ERP 匯出 3 份 CSV，手動貼到 Excel 做樞紐分析，再截圖發到 Line 群組。
* **問題：** 耗時且容易貼錯欄位；若助理請假，當天就沒報表看。
* **觸發點：** 業務經理要求報表頻率從「每天一次」改成「每半天一次」，人工已無法負荷。

#### 2. Goals & ROI (效益)

* **目標：** 透過 Python Script 自動撈取 DB，定時產出報表並發送通知。
* **預期效益：**
* **省時：** 每天節省助理 1.5 小時 (每月約 30 工時)。
* **正確性：** 100% 排除人工複製貼上的錯誤。



#### 3. Scope (邊界)

| ✅ In Scope (本次範圍) | ❌ Out of Scope (不包含) |
| --- | --- |
| **自動撈取 Sales DB** | **修改 ERP 內的資料** (只讀不寫) |
| **產出 Excel 報表** (依指定格式) | **互動式 Dashboard** (不做 BI 網頁) |
| **Email 自動寄送** (給業務主管) | **Line Notify 通知** (資安考量暫緩) |
| **排程：每日 09:00 & 14:00** | **即時查詢功能** |

#### 4. Solution (技術方案)

* **技術：** Python (Pandas) + SQL + Jenkins (排程)。
* **輸入：** 業務部提供的 Excel 模板 (Template.xlsx)。
* **輸出：** 填好數據的 Excel 檔，透過 SMTP Server 寄出。

#### 5. Maintenance (維運條款)

* **故障分級：** 若機器人掛點，請業務助理**暫時切回人工製作** (不保證 1 小時內修復)。
* **資料錯誤：** 若源頭 ERP 資料有誤，本工具不負責修正。
* **需求變更：** 若 Excel 格式要改，需提早 3 天通知開發者修改 Code。
